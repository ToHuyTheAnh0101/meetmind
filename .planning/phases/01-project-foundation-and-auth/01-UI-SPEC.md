# Phase 1 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Scope Note

Phase 1 contains exactly two screens with visible UI:

1. **Login page** (`/login`) — the only screen with meaningful design decisions
2. **Auth callback** (`/auth/callback`) — a loading/processing screen; full-screen centered spinner only, no chrome

All design decisions below are driven by the login page. The callback route is a degenerate case: one element, one state.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn/ui (CLI 4.1.0) |
| Preset | none — initialized interactively via `npx shadcn init` during Plan 1; style: New York, base color: Neutral, CSS variables: yes |
| Component library | Radix UI (via shadcn) |
| Icon library | lucide-react 1.0.1 |

**shadcn Gate result:** `components.json` does not exist yet — project is pre-scaffold. shadcn will be initialized in Phase 1 Plan 1. Registry safety gate applies once initialized.

**shadcn init recommended flags:**
```
Style:        New York
Base color:   Neutral
CSS variables: Yes
```
New York style is preferred over Default for its tighter, more professional proportions. Neutral base avoids blue/zinc bias and pairs cleanly with white surfaces.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap inside the Google button |
| sm | 8px | Stacked element gap (heading → subheading) |
| md | 16px | Default internal card padding |
| lg | 24px | Card padding (top/bottom) |
| xl | 32px | Card padding (left/right) on desktop |
| 2xl | 48px | Vertical space between logo and card |
| 3xl | 64px | Minimum viewport top padding on mobile |

Exceptions: Google button minimum touch target is 44px tall (accessibility — WCAG 2.5.5). This is not a spacing token; it is a minimum height constraint on the `<Button>` component.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FFFFFF` (white) | Page background, card background |
| Secondary (30%) | `#F9FAFB` (gray-50) | Card surface on login page (subtle lift from page background) |
| Accent (10%) | `#18181B` (zinc-900) | Primary button fill ("Sign in with Google") |
| Destructive | `#EF4444` (red-500) | Auth error message text; not used for actions in Phase 1 |

Accent reserved for: **the "Sign in with Google" button only** in Phase 1. No other element uses the accent fill.

Border color: `#E4E4E7` (zinc-200) — card border and button border (ghost variant if used).

Text primary: `#09090B` (zinc-950) — headings and body text.

Text muted: `#71717A` (zinc-500) — subheading copy below the app name.

**Semantic note:** The dark button (`zinc-900` fill, white label) is deliberate. It reads as a professional, neutral CTA rather than a "Google blue" button. The Google logo SVG inside the button provides brand recognition without requiring a colored button. This aligns with the "subtle colors, professional" direction.

Dark mode: not applicable — explicitly out of scope per REQUIREMENTS.md.

## Screen Specifications

### Screen 1: Login Page (`/login`)

**Layout:** Full-viewport centered layout. Single card vertically and horizontally centered. Responsive — card is full-width on mobile with 16px horizontal margin, max-width 400px on desktop.

```
┌─────────────────────────────────────────┐
│                                         │
│                                         │  ← 48px top space minimum
│         [MeetMind wordmark]             │
│                                         │  ← 48px gap
│  ┌───────────────────────────────────┐  │
│  │ Welcome to MeetMind               │  │  ← 28px / semibold
│  │ Sign in to manage your meetings   │  │  ← 16px / regular / zinc-500
│  │                                   │  │
│  │  [G] Sign in with Google          │  │  ← Button: full-width, 44px tall
│  └───────────────────────────────────┘  │
│                                         │
└─────────────────────────────────────────┘
```

**Card dimensions:** max-w-[400px], padding 24px vertical / 32px horizontal (desktop), 24px all sides (mobile).

**Button:** Full width within card (`w-full`). Height: min 44px. Icon: Google "G" SVG, 20px. Label: "Sign in with Google", 14px semibold.

**Error state:** When auth fails (e.g., backend returns an error on callback redirect), show an inline error message below the button inside the card. See Copywriting Contract for exact copy. Error text: 14px regular, red-500 (`#EF4444`).

**App name/wordmark:** "MeetMind" displayed as plain text above the card, 20px semibold, zinc-950. No logo image in Phase 1.

**States:** The login page has two states — idle and error. There is no loading state on the login page itself (the button click immediately navigates away via `window.location.href`; no spinner needed on the button).

---

### Screen 2: Auth Callback (`/auth/callback`)

**Layout:** Full-viewport centered. Single element: spinner + status text.

```
┌─────────────────────────────────────────┐
│                                         │
│           [spinning icon]               │
│           Signing you in…               │
│                                         │
└─────────────────────────────────────────┘
```

**Spinner:** `<Loader2>` from lucide-react, 24px, `animate-spin`, color zinc-400.

**Text:** "Signing you in…" — 16px regular, zinc-500, 8px below the spinner.

**Duration:** The callback page is transient (< 1s in normal conditions). No additional UI chrome needed. No back button, no header, no card.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Sign in with Google |
| App name (above card) | MeetMind |
| Card heading | Welcome to MeetMind |
| Card subheading | Sign in to manage your meetings |
| Callback status | Signing you in… |
| Auth error heading | Sign-in failed |
| Auth error body | We couldn't complete your Google sign-in. Please try again. |
| Auth error retry action | Try again (links back to `/login` or re-triggers the Google flow) |

Empty state: not applicable in Phase 1 (no data-driven screens).

Destructive actions: none in Phase 1. No confirmation dialogs required.

**Copy rules:**
- Use sentence case, not title case, for all copy except the app name "MeetMind"
- No exclamation marks
- Error messages describe the problem and offer a next step — never raw error codes or stack traces
- The callback page uses an ellipsis ("…") on "Signing you in…" to signal ongoing action without a percentage or countdown

## Accessibility Baseline

- "Sign in with Google" button must have visible focus ring (shadcn Button includes this by default via Radix)
- Spinner on callback page must have `aria-label="Signing you in"` for screen readers
- Color contrast: zinc-950 on white = 19.1:1 (WCAG AAA); zinc-500 on white = 4.6:1 (WCAG AA)
- Button minimum touch target: 44px height (declared in Spacing exceptions above)
- Error message must be associated with the button region via `role="alert"` so screen readers announce it
