<objective>
Implement complete authentication flow with Google OAuth, three-state auth management, and protected routes
</objective>

<purpose>
Build the authentication system that enables users to sign in with Google OAuth, maintain session state across page refreshes, and access protected routes. This plan implements the AuthContext with three-state status (loading/authenticated/unauthenticated), the login page with OAuth redirect, the callback handler that stores tokens and validates them, and the ProtectedRoute component that guards authenticated pages. Without this, users cannot authenticate or access the dashboard.
</purpose>

- LoginPage with shadcn/ui Card and "Sign in with Google" button
- AuthCallbackPage that handles OAuth callback, stores token, strips from history, validates with backend
- ProtectedRoute component that wraps authenticated routes
- TypeScript types for API responses
- Complete auth flow working end-to-end with backend
</output>

  **Step 2: Create AuthContext (frontend/src/features/auth/AuthContext.tsx)**
  - Import React, createContext, useContext, useState, useEffect from 'react'
  - Import useNavigate from 'react-router-dom'
  - Import getToken, setToken, clearToken, isTokenExpired from '@/lib/tokenStorage'
  - Import apiClient from '@/lib/apiClient'
  - Import type { User } from '@/types/api'

  - Define AuthStatus type: 'loading' | 'authenticated' | 'unauthenticated'
  - Define AuthState interface with fields: status (AuthStatus), user (User | null), logout (() => void)

  - Create AuthContext with type AuthState | null, initialize to null

  - Create AuthProvider component accepting children prop
  - Inside AuthProvider:
    - Initialize status state with 'loading' (CRITICAL - prevents auth flicker per D-02, D-03)
    - Initialize user state with null
    - Get navigate from useNavigate

  - Implement bootstrap verification useEffect:
    - On mount, call getToken()
    - If no token or isTokenExpired(token) is true:
      - Call clearToken()
      - Set status to 'unauthenticated'
      - Return early
    - If token exists and valid:
      - Call apiClient.get('/auth/verify')
      - On success: extract user from response.data.user, setUser(user), setStatus('authenticated')
      - On error: call clearToken(), setStatus('unauthenticated')
    - Empty dependency array (runs once on mount per D-03)

  - Implement auth:logout event listener useEffect:
    - Add event listener for 'auth:logout' on window
    - On event: setUser(null), setStatus('unauthenticated'), navigate('/login', { replace: true })
    - Cleanup: remove event listener on unmount
    - Dependency array: [navigate]

  - Implement logout function:
    - Call clearToken()
    - Set user to null
    - Set status to 'unauthenticated'
    - Call navigate('/login', { replace: true })

  - Return AuthContext.Provider with value object containing status, user, logout
  - Wrap children in provider

  - Create useAuth hook:
    - Call useContext(AuthContext)
    - Throw error if context is null ("useAuth must be used within AuthProvider")
    - Return context

  - Export AuthProvider and useAuth

  **Step 3: Wire up AuthProvider in App.tsx**
  - Import AuthProvider from '@/features/auth/AuthContext'
  - Wrap the Routes element with AuthProvider (inside BrowserRouter, outside Routes)
  - Pattern: <BrowserRouter><AuthProvider><Routes>...</Routes></AuthProvider></BrowserRouter>

  **Anti-patterns to avoid:**
  - Do NOT initialize status as 'false' or 'null' - must be 'loading' to prevent auth flicker
  - Do NOT call useNavigate outside React component - AuthProvider is a component so this is OK
  - Do NOT skip the auth:logout event listener - 401 interceptor needs this
  - Do NOT call /auth/verify without checking token expiry first - saves unnecessary API calls
  </action>

  **Step 2: Implement AuthCallbackPage (frontend/src/features/auth/AuthCallbackPage.tsx)**
  - Import React, { useEffect, useState } from 'react'
  - Import { useNavigate } from 'react-router-dom'
  - import { setToken, clearToken } from '@/lib/tokenStorage'
  - import apiClient from '@/lib/apiClient'
  - import { Loader2 } from 'lucide-react'

  - Create AuthCallbackPage component
  - Initialize state: error (string | null), isLoading (boolean, default true)
  - Get navigate from useNavigate

  - Implement useEffect for callback handling:
    - Read URL params synchronously: new URLSearchParams(window.location.search)
    - Get token from params.get('token')
    - If no token:
      - Set error state to "No token received from OAuth callback"
      - Set isLoading to false
      - Return early

    - Strip token from browser history immediately:
      - Call window.history.replaceState({}, document.title, window.location.pathname)
      - This prevents JWT from appearing in Referer headers (per D-09)

    - Store token: call setToken(token)

    - Call apiClient.get('/auth/verify') to validate token
    - On success:
      - Navigate to '/' with replace: true
    - On error:
      - Call clearToken()
      - Set error state to "Authentication failed. Please try again."
      - Set isLoading to false

    - Empty dependency array (runs once on mount)

  - Return JSX with loading/error states:
    - If isLoading:
      - Centered div with flex, min-h-screen, items-center, justify-center
      - Show Loader2 icon spinning (animate-spin)
      - Text: "Signing you in..."
    - If error:
      - Centered div with flex, min-h-screen, items-center, justify-center
      - Show error message in red text
      - Button with "Try again" that navigates to '/login'

  - Export AuthCallbackPage

  **Anti-patterns to avoid:**
  - Do NOT use fetch() for OAuth redirect - must use window.location.href
  - Do NOT read token from React Router's useSearchParams() - use window.location.search directly
  - Do NOT skip history.replaceState - token will leak in browser history
  - Do NOT show error before stripping token - strip first, then handle errors
  - Do NOT navigate without replace: true - prevents back button from returning to callback
  </action>
